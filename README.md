# CS 340 Portfolio Journal 

## Reflection

### Writing Maintainable, Readable, and Adaptable Programs

Writing maintainable, readable, and adaptable programs requires clear structure, consistent naming conventions, and modular design. In this course, the development of the CRUD Python module in Project One played a key role in achieving these goals. By separating database operations into a dedicated module, the code became easier to manage and reuse.

The main advantage of this approach was modularity. Instead of embedding database logic directly into the dashboard code, the CRUD module acted as an intermediary layer. This made the system more organized and easier to debug, since each component had a clearly defined responsibility. Additionally, this structure improved readability because the dashboard code focused only on user interface functionality, while the CRUD module handled database interactions.

In the future, this CRUD module could be reused in other applications that require database connectivity. For example, it could be adapted for use in web applications, APIs, or data analysis tools where consistent database access patterns are needed.

### Problem-Solving Approach as a Computer Scientist

When approaching this project, I followed a structured problem-solving process. First, I analyzed the requirements provided by Grazioso Salvare to understand the data needs and expected functionality. Then, I designed the database queries and structure to meet those requirements before implementing the dashboard interface.

This approach differed from earlier assignments because it involved integrating multiple components, including the database, backend logic, and frontend dashboard. Instead of solving isolated problems, I had to think about how different parts of the system interact with each other.

In future projects, I would continue using strategies such as breaking down complex requirements into smaller tasks, designing systems before coding, and testing each component individually. These techniques help ensure that the final system is both functional and scalable.
